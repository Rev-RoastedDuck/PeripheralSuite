## 一、文件与版权规范
1. **文件头部注释**
   - 必含信息：创建时间、作者、开源地址（Github）、版权声明，清晰追溯代码归属和版本；
   - 格式统一：使用 `/* */` 块注释，排版规整，关键信息（如作者、版权）单独成行，可读性强。
2. **头文件保护**
   - 使用 `#ifndef PY32F303K28U6_SPI_RRD_H_` + `#define` + `#endif` 防止头文件重复包含，宏命名规则：`芯片型号_模块名_作者标识_H_`。
3. **调试代码隔离**
   - 用 `#define OPEN_SPI_TEST (0)` 控制调试代码（`spi_test_rrd`）的编译，正式发布时只需改宏值即可屏蔽调试逻辑，不污染生产代码。
4. **示例**：
    ```c
    /*
    * 创建时间: 2026-02-24
    * 作者: XXX
    * 开源地址: https://github.com/xxx/py32f030k28u6-lib
    * 版权声明: MIT License
    */
    ```

## 二、命名规范
### 1. 统一后缀标识
- 所有自定义标识符（枚举、结构体、函数、宏）均以 `_rrd` 结尾（作者标识/项目标识），避免与系统/其他模块命名冲突；
- 示例：`SPI_RRD`、`spi_init_rrd`、`SPI_CONFIG_DEFAULT_RRD`。

### 2. 类型与实例区分
- 枚举/结构体类型：大写蛇形命名 + 大写后缀`_RRD`，同时定义小写别名；
- 函数/变量：小写 + 下划线分隔（`spi_get_miso_pin_info_rrd`）。
    - 全局变量：`g_` + 大写蛇形命名；
    - 宏定义：`大写蛇形命名。
- 示例：`SPI_INDEX_RRD`、`spi_index_rrd`、`g_SPI_INTERFACE_RRD`。

### 3. 功能语义化命名
- 函数名 = 模块 + 动作 + 功能 + 标识：`spi_write_buff_block_rrd`（SPI模块+写+缓冲区+阻塞方式+RRD标识）。
- 私有函数、回调函数、私有成员变量、函数指针-typedef，需要在函数名前添加`_`前缀；
- 枚举值 = 模块 + 引脚类型 + 引脚号：`SPI1_MISO_PA0`（SPI1+MISO引脚+PA0），见名知意。

### 4. 后缀

## 三、代码组织结构规范
1. **分层设计（PRIVATE/PUBLIC/INIT-DEINIT/DEBUG）**
   - `PRIVATE` 段：静态内联函数（`static inline`），仅模块内部使用，对外隐藏实现细节；
   - `PUBLIC` 段：对外暴露的核心接口（如 `spi_read_buff_rrd`/`spi_write_buff_rrd`），作为模块的“对外契约”；
   - `INIT-DEINIT` 段：用于SPI设备的初始化与反初始化相关逻辑，包含`spi_init_rrd` 初始化函数、`spi_deinit_rrd` 反初始化函数，以及接口结构体（如 `g_SPI_INTERFACE_RRD`）的定义；
   - `DEBUG` 段：调试函数/中断服务例程（如 `spi_test_rrd`/`SPI2_IRQHandler`），通过条件编译隔离，不影响生产环境的核心逻辑。
2. **功能分组注释**
   - 使用 `/** \addtogroup write ** \{ */` 对读写功能分组，配合 `/** \} */` 闭合，符合Doxygen文档规范，便于自动生成文档；
   - 关键代码块（如GPIO初始化、SPI配置）用 `{// 注释}` 包裹，逻辑边界清晰。
3. **函数职责单一**
   - 每个函数只做一件事：`spi_get_miso_pin_info_rrd` 仅获取MISO引脚信息，`spi_write_buff_block_rrd` 仅实现阻塞写，不混杂其他逻辑，便于维护和调试。

## 四、语法与编码风格规范
1. **花括号与换行规范**
   - 函数开头的花括号另起一行。
   - 其他关键字（`if`、`for`、`while`、`switch`）的花括号与关键字在同一行，且花括号与关键字（或括号）之间不要增加空格。
   - `else`、`else if` 必须单独新起一行。
2. **空格与间距规范**
   - 逗号 `,` 后要有一个空格隔开。
   - 运算符号（如 `=`、`==`、`+`、`-`、`>`、`<` 等）左右各保留一个空格。
3. **指针命名规范**
   - 指针符号 `*` 靠近参数名/变量名。示例：`void *self`。
4. **代码逻辑优化**
   - 对于退出作用域的 `if` 判断（如参数校验、错误拦截），如果内部只有返回函数，建议写成一行。示例：`if(NULL == self) return 1;`
   - 高频调用的小函数（如引脚匹配检查、读写底层逻辑）用 `static inline` 修饰，减少函数调用开销，提升嵌入式系统执行效率。
5. **返回值规范**
   - 错误码语义化：`return 2` 对应“DMA通道无空闲”，`return 4` 对应“MISO引脚信息获取失败”，可通过返回值快速定位问题。
   
## 五、接口设计规范
1. **句柄（self）机制**
   - 核心结构体 `SPI_RRD` 作为“对象句柄”，所有接口函数均以 `self` 为第一个参数，实现“一个句柄对应一个SPI设备实例”，支持多SPI设备管理；
   - 句柄内部分离 `config`（配置）、`params`（运行时参数）、`interface`（函数接口）。
2. **接口抽象化**
   - 定义函数指针结构体 `SPI_INTERFACE_RRD`，封装读写/中断处理接口，底层实现（阻塞/中断/DMA）与上层调用解耦；
   - 上层只需调用 `self->interface->read/write`，无需关心底层是阻塞、中断还是DMA方式，便于后续替换实现。
3. **配置默认值**
   - 定义 `SPI_CONFIG_DEFAULT_RRD` 宏，提供SPI默认配置，初始化时直接复用，减少重复的配置代码，同时保证默认配置的一致性。

## 六、文件管理
1. **分层分级**：按“数字前缀+功能职责”划分顶层目录，层级间单向依赖（上层调用下层，不反向依赖）；
2. **职责单一**：每个目录/文件仅承担一类功能；
3. **命名统一**：目录/文件命名采用小写蛇形，模块标识统一加`_rrd`后缀，第三方库保持原生命名；
4. **接口与实现分离**：核心模块均区分`interface/`（接口定义）和`src/`（具体实现），降低耦合。
